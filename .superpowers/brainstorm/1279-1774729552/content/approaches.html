3 подхода к архитектуре

Выбери подход, который ближе всего к твоему видению

A

Canvas/WebGL + SPA

Стек: React/Next.js + PixiJS или Three.js для рендера таймлайна

Суть: Таймлайн рисуется на canvas — полный контроль над анимациями, зумом, переходами. Десятки тысяч событий без тормозов. Данные из Wikidata кэшируются в SQLite/JSON на сервере.

Плюсы

  • Максимальная производительность при большом количестве событий
  • Полный контроль над визуалом и анимациями
  • Плавный зум от тысячелетий до дней

Минусы

  • Сложнее в разработке — canvas не даёт бесплатной доступности
  • Текст на canvas менее чёткий, чем DOM
  • SEO ограничен
B

DOM + CSS Animations

Стек: React/Next.js + Framer Motion + CSS transforms для зума

Суть: Таймлайн построен из DOM-элементов с CSS-трансформациями. Виртуализация для производительности (рендерим только видимые события). Анимации через Framer Motion.

Плюсы

  • Проще в разработке — стандартный React
  • Чёткий текст, доступность, SEO
  • Framer Motion даёт красивые анимации из коробки

Минусы

  • Лимит ~1000-2000 DOM-элементов одновременно
  • Зум менее плавный на больших масштабах
  • Сложнее реализовать "бесконечный" зум
C

Гибрид: Canvas фон + DOM оверлей

Стек: React/Next.js + HTML Canvas для оси/линий + DOM для карточек событий + Framer Motion

Суть: Canvas рисует ось времени, линии связей и фоновые эффекты. Карточки событий — DOM-элементы поверх canvas. Лучшее от обоих миров.

Плюсы

  • Красивый фон + чёткие читаемые карточки
  • Canvas для тяжёлой графики, DOM для интерактива
  • Хороший баланс производительности и удобства

Минусы

  • Нужно синхронизировать два слоя (canvas + DOM)
  • Чуть сложнее архитектура
