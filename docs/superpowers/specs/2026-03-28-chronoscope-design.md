# ChronoScope — Интерактивный таймлайн мировой истории

## Обзор

Веб-приложение с интерактивным горизонтальным таймлайном, показывающим мировые исторические события, даты и персоналии. Главная ценность — возможность увидеть параллельность событий (например, что происходило в мире при жизни Пушкина). Все события ведут на Wikipedia.

## Стек

- **Framework:** Next.js (App Router) + TypeScript
- **Рендер таймлайна:** гибрид Canvas 2D (ось, линии, glow-эффекты) + DOM (карточки событий, полосы жизни, UI)
- **Анимации:** Framer Motion (DOM-элементы) + requestAnimationFrame (canvas)
- **Данные:** Wikidata SPARQL API + Wikipedia REST API
- **Деплой:** Vercel (статика + edge functions для кэширования Wikidata)

## Источник данных

### Wikidata SPARQL

Запросы к `query.wikidata.org` для получения структурированных данных:
- События: войны, революции, открытия, произведения, договоры
- Персоналии: правители, учёные, писатели, художники, полководцы
- Каждая сущность: название, дата (начало/конец), категория, координаты (опционально), Wikidata ID

### Wikipedia REST API

- Краткое описание (экстракт первого абзаца) для карточки события
- Изображение (если доступно) для развёрнутой карточки

### Стратегия загрузки

| Масштаб | Видимый период | Данные | Источник |
|---------|---------------|--------|----------|
| 7000 лет | Вся история | ~50 эпох и цивилизаций | SSG (предзагружено в JSON при билде) |
| 1000 лет | Тысячелетие | ~200 крупных событий | SSG |
| 100 лет | Век | ~500 ключевых событий и персон | SSG |
| 10 лет | Декада | ~50-100 событий в окне | Lazy-fetch из Wikidata |
| 1 год | Год | Конкретные даты, месяцы | Lazy-fetch из Wikidata |

Базовый набор (~5000 ключевых событий) собирается при билде через SPARQL и хранится в статическом JSON. При зуме до детального уровня — дополнительные запросы к Wikidata с кэшированием в localStorage (stale-while-revalidate).

### Структура данных

```typescript
interface HistoryEvent {
  id: string;            // Wikidata ID (e.g. "Q362")
  title: string;         // "Отечественная война 1812 года"
  dateStart: string;     // ISO date "1812-06-24"
  dateEnd?: string;      // ISO date или undefined для точечных событий
  category: Category;    // "politics" | "science" | "culture" | "religion" | "technology" | "person"
  subcategory?: string;  // "war", "revolution", "discovery", etc.
  geo?: [number, number]; // [lat, lng] опционально
  wikiUrl: string;       // URL на Wikipedia
  description?: string;  // Краткое описание (подгружается лениво)
  imageUrl?: string;     // Изображение (подгружается лениво)
}

type Category = "politics" | "science" | "culture" | "religion" | "technology" | "person";
```

## Интерфейс

### Общая компоновка

Тёмная тема (фон #0a0a1a) с неоновыми акцентами, цветовая кодировка по категориям:
- Политика: синий (#818cf8)
- Наука: фиолетовый (#c084fc)
- Культура: красный (#f87171)
- Религия: жёлтый (#fbbf24)
- Технологии: голубой (#22d3ee)
- Люди: зелёный (#34d399)

### Слои

1. **Canvas-слой (фон):** ось времени с градиентной подсветкой, точки событий с glow-эффектом, линии связей, фоновые частицы
2. **DOM-слой (поверх canvas):** карточки событий, полосы жизни персоналий, UI-элементы (хедер, поиск, фильтры, зум-контроли)

### Хедер

- Логотип "CHRONOSCOPE" слева
- Поиск по имени/событию/году (автокомплит)
- Фильтры-переключатели по категориям (множественный выбор)

### Ось времени

- Горизонтальная линия с градиентом, точки событий на оси с glow
- Метки годов/веков в зависимости от масштаба
- Полосы жизни персоналий — горизонтальные полупрозрачные полосы над/под осью

### Карточки событий

- Компактный вид: категория, заголовок, дата, страна
- Развёрнутый вид (по клику): описание из Wikipedia, изображение, связанные события/персоны, кнопка "Читать на Wikipedia"

### Нижняя панель

- Быстрые переходы по эпохам: Древний мир → Античность → Средневековье → Возрождение → Новое время → XX век → Современность
- Ползунок масштаба

## Взаимодействие

### Навигация

- **Scroll колёсиком:** зум (плавная анимация с перегруппировкой событий)
- **Drag мышью:** панорамирование по оси времени
- **Pinch (тачскрин):** зум на мобильных
- **Клик на карточку:** раскрытие деталей
- **Клик на полосу жизни:** подсветка всех связанных событий эпохи

### Поиск

- Ввод текста → автокомплит из предзагруженных данных + Wikidata search API
- Выбор результата → таймлайн плавно анимируется к нужному периоду
- Для персоналий: центрирование на годах жизни + показ параллельных событий

### Фильтры

- Кнопки-переключатели категорий в хедере
- Множественный выбор (можно включить несколько категорий)
- Плавное появление/исчезновение карточек (Framer Motion AnimatePresence)

## Производительность

- **Виртуализация DOM:** рендерим только карточки в видимой области ±буфер
- **Canvas оптимизация:** перерисовка только при изменении viewport
- **Предзагруженные данные:** ~5000 событий в статическом JSON (~1-2 MB gzipped)
- **Lazy-fetch:** детальные данные подгружаются только при глубоком зуме
- **Кэширование:** localStorage для Wikidata-ответов, stale-while-revalidate

## За пределами MVP

Не входит в первую версию:
- Карта мира с событиями на географической привязке
- Аккаунты пользователей / сохранение состояния
- Многоязычность (MVP только на русском)
- Создание пользовательских таймлайнов
- Сравнение двух периодов side-by-side
- Бэкенд с базой данных (всё на клиенте + Wikidata API)

## Тестирование

- **Unit:** вычисление позиций на оси, фильтрация, парсинг Wikidata-ответов
- **Integration:** загрузка данных, поиск, зум-навигация
- **Visual:** Storybook для компонентов карточек, полос жизни
- **E2E:** базовый сценарий — открыть → зум → клик на событие → ссылка на Wiki

## Ошибки и edge cases

- **Wikidata недоступна:** показываем предзагруженные данные, уведомление "Дополнительные данные временно недоступны"
- **Неточные даты:** некоторые древние события имеют только год или век — отображаем с соответствующей точностью
- **Пустой период:** при зуме в период без событий — сообщение "Нет событий для выбранных фильтров"
- **Мобильные устройства:** адаптивный лейаут, touch-события для навигации
